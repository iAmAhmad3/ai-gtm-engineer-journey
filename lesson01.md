# Clay 101 - Lesson 01

**Topic:** Introduction to Clay & the FETE Framework

---

## Objective

This lesson introduces **Clay** and its core workflow methodology, the **FETE Framework**. Instead of approaching Clay as a blank workspace with many disconnected features, every workflow can be understood through four repeatable steps:

> **Find → Enrich → Transform → Export**

This framework is the foundation for building workflows in Clay.

---

## Why the FETE Framework?

One of the biggest challenges for new users is the **"blank canvas problem."** The FETE Framework provides a structured approach that simplifies workflow creation and helps users think systematically about data processing.

---

## The FETE Framework

### 1. Find

The first step is collecting your initial dataset.

You can populate a Clay table by:

- Importing contacts or companies from a CRM
- Using Clay's native search tools
- Importing CSV files
- Scraping websites
- Connecting external data sources

**Goal**

Build the initial list of companies or people you want to work with.

---

### 2. Enrich

Once your initial dataset is available, enrich it with additional information.

Enrichment can include:

- AI-generated insights
- Company information
- Contact details
- Social profiles
- Technology stack
- Funding information
- Data from third-party providers

**Goal**

Convert basic records into valuable, information-rich datasets.

---

### 3. Transform

Raw data is rarely ready for use. The Transform stage prepares the data by making it clean and consistent.

Common transformations include:

- Cleaning messy data
- Standardizing formats
- Applying formulas
- Using AI to restructure content
- Filtering unwanted records
- Creating calculated fields

**Goal**

Prepare the data for automation, analysis, or outreach.

---

### 4. Export

After processing the data, export it to the tools where it will be used.

Common destinations include:

- HubSpot
- Salesforce
- Google Sheets
- Apollo
- Email sequencing platforms
- Other CRM systems

**Goal**

Move processed and actionable data into operational systems.

---

## Workflow Overview

```text
Find
   ↓
Enrich
   ↓
Transform
   ↓
Export
```

---

## Real-World Example

Suppose you want to build a prospect list for B2B SaaS companies.

### Find

Search for SaaS companies with 10–200 employees.

↓

### Enrich

Retrieve:

- Founder names
- Email addresses
- LinkedIn profiles
- Company funding
- Technology stack

↓

### Transform

- Standardize company names
- Generate personalized first lines with AI
- Score leads based on your Ideal Customer Profile (ICP)

↓

### Export

Send qualified leads to:

- HubSpot
- Apollo
- Instantly
- Smartlead

---

## Key Takeaways

- Every Clay workflow follows the same four-step process.
- The FETE Framework provides a simple mental model for building workflows.
- Instead of focusing on individual features, identify which stage of the FETE process you are working on.
- Mastering this framework will make advanced workflows much easier to understand.

## Key Terms

| Term | Description |
|------|-------------|
| FETE | Find → Enrich → Transform → Export |
| Find | Collect the initial dataset |
| Enrich | Add additional information to the data |
| Transform | Clean, organize, and standardize the data |
| Export | Send processed data to external tools |

---

## Lesson Summary

The FETE Framework is the core workflow model used throughout Clay.

Rather than treating Clay as a collection of separate features, every workflow can be broken down into four consistent stages:

> **Find → Enrich → Transform → Export**

Understanding this framework first will make future lessons easier to follow and help you design more effective workflows.
